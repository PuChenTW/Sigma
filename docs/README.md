# Documentation

Docs are split by decision type.

## Product

Product docs describe what the product is, where it is today, and how it should evolve.

- [Product North Star](product/north-star.md): final product shape, user problem, target experience, capabilities, and product boundaries.
- [Status And Roadmap](product/status-and-roadmap.md): current repository status, limitations, useful commands, and near-term planning.
- [Product Evolution Plan](product/evolution-plan.md): staged path from the MVP toward the prototype-like product experience.

## Technical

Technical docs describe how the system is structured and how interfaces behave.

- [Architecture](technical/architecture.md): runtime, components, domain model, boundaries, persistence, and technical direction.
- [API Design](technical/api.md): current API surface and API rules.
